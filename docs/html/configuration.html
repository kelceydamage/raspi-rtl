Configuration¶

rtl.transport.conf.configuration.PIDFILES¶

Location to store pid files.

  • Default: '~/var/run/'

Type

str

rtl.transport.conf.configuration.TASK_LIB¶

Module name for tasks library.

  • Default: 'rtl.tasks.*'

Type

str

rtl.transport.conf.configuration.DEBUG¶

True to enable debug printing, else False to prevent.

  • Default: False

Type

bool

rtl.transport.conf.configuration.PROFILE¶

True to enable timings, else False to prevent.

  • Default: False

Type

bool

rtl.transport.conf.configuration.STARTING_PORT¶

Port block for workers to consume.

  • Default: 10000

Type

int

rtl.transport.conf.configuration.TASK_WORKERS¶

The number of task workers to spawn.

  • Default: 3

Type

int

rtl.transport.conf.configuration.PLOT_WORKERS¶

The number of plot workers to spawn.

  • Default: 1

Danger

Setting this above 1 could have unexpected behaviors.

Type

int

rtl.transport.conf.configuration.PLOT_LISTEN¶

The port on which to server plots.

  • Default: 5006

Type

int

rtl.transport.conf.configuration.PLOT_ADDR¶

The IP address that the plot worker will listen on.

  • Default: '0.0.0.0'

Type

str

rtl.transport.conf.configuration.RESPONSE_TIME¶

Controls the rate at which tasks are sent to the workers, and in doing so, the size of the queue.

Examples

1000 req @0.01 = ~100 tasks per queue.

1000 reg @0.001 = ~10 tasks per queue.

  • Default: 0.005

Type

float

rtl.transport.conf.configuration.RELAY_LISTEN¶

The IP address that the relay worker will listen on.

  • Default: '0.0.0.0'

Type

str

rtl.transport.conf.configuration.RELAY_ADDR¶

The IP address that relay client will connect to.

  • Default: '127.0.0.1'

Type

str

rtl.transport.conf.configuration.RELAY_RECV¶

The recieve port on the relay worker.

  • Default: 19000

Type

int

rtl.transport.conf.configuration.RELAY_SEND¶

The send port on the relay worker.

  • DefaultL 19001

Type

int

rtl.transport.conf.configuration.RELAY_PUBLISHER¶

The publisher port on the relay worker.

  • Default: 19300

Type

int

rtl.transport.conf.configuration.CHUNKING¶

Enable chunking service on the relay worker.

Important

Chunking determines if and how much the router breaks up queues in order the better balance worker loads.

Examples

A setting of 10 will break up all queues int ~ 10 tasks per worker. This will negativly affect response time since it adds delay at the router, and extra network activity.

Note

RESPONSE_TIME and CHUNKING should be balanced to get an Optimal throughput and worker load balance.

  • Default: True

Type

bool

rtl.transport.conf.configuration.CHUNKING_SIZE¶

The desired size of each chunk.

  • Default: 1000000

Type

int

rtl.transport.conf.configuration.CACHE_PATH¶

The path to store the cache file.

  • Default: '/tmp/transport'

Type

str

rtl.transport.conf.configuration.CACHE_MAP_SIZE¶

The size of the cache (memory allocation) for the task cache.

Note

The default value is 512MB for embedded systems

  • Default: 512*1024**2

Type

int
